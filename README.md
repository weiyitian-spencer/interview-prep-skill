# Interview Prep Skill — 面试项目经历准备

## 简介

这是一个帮助产品/运营岗求职者把真实工作经历系统转化为面试素材的 AI Skill，覆盖从素材整理到实战演练的完整链路，支持中英文双版本输出。

## 功能说明

### 四个阶段

```
阶段一：元项目库构建  →  衔接：能力映射  →  阶段二：JD 适配改写  →  阶段三：追问准备  →  阶段四：Mock Interview
   (一次性，可复用)       (动态更新)           (每投递一个岗位做一次)    (基于改写后的项目)      (实战压力测试)
        ↑                     ↑                                               │
        └─────────────────────┴───────── 追问中挖出新细节 / 新 JD 补充维度 ───┘
```

- **阶段一：元项目库构建** — 逐项挖掘 STAR 要素，整理成包含基本信息、能力覆盖标签、关键决策、核心挑战与方法论沉淀的完整项目底稿
- **能力映射** — 维护能力维度总表、缺口预警、跨 JD 频次统计，随 JD 积累动态更新；项目与能力的对应关系直接查各项目底稿的能力标签
- **阶段二：JD 适配改写** — 针对具体岗位调整叙述角度，输出：陈述稿（中英文）、JD 映射说明、自我介绍（中英文）
- **阶段三：追问准备** — 生成高概率追问题库，打磨应答，纠正"听起来不像亲身经历"的破绽；结束后回溯补充元项目库
- **阶段四：Mock Interview** — 模拟真实面试压力，AI 扮演面试官全程不提示，结束后给结构化反馈

### 设计理念

核心是"一次建库、持续迭代"——元项目库是贯穿整个求职周期的可复用资产，不是一次性用完的准备材料。每投递一个新 JD 只需在已有素材基础上调整角度；追问打磨中挖出的新细节反哺元项目库，JD 积累多了反向扩充能力维度，整个库越用越完整。流程设计上强调执行落地：AI 在每个关键节点主动判断进度、输出格式对齐的可粘贴片段，把用户的手动操作压到最低。

## 交付内容

### 过程产物（每个阶段结束后输出可粘贴片段）
- **元项目底稿** — 包含基本信息、能力覆盖（强/中/弱）、STAR 要素、核心挑战与方法论沉淀、反思、追问补充
- **JD 适配陈述稿** — 中英文各一版，电梯版（30秒关键词串 + 口语陈述）+ 完整版（2-3分钟）
- **JD 映射说明** — 能力点 → JD 要求 → 一句话连接逻辑
- **自我介绍** — 中英文各一版（1分钟），"我是谁 → 相关经历主线 → 为什么在这里"
- **追问题库 + 应答要点**

### 最终文档（按岗位生成 Word 文档）
```
一、岗位理解
二、自我介绍（中英文各一版）
三、项目经历（元项目完整版 + 电梯版 + 完整版 + JD 映射）
四、行为面试题库（按题型归类）
五、案例题准备
六、英文面试专项
```

## Job Vault 结构

推荐在使用前建立以下文件结构，用于跨会话持久化：

```
Job Vault/
├── 元项目库.md       # 每个项目一份，STAR 结构 + 能力标签 + 追问补充
├── 能力映射图.md     # 维度总表 + 缺口预警 + 跨 JD 频次统计
└── JD 存档.md        # 投过的每份 JD 原文 + 解析出的关键能力词
```

## 使用方式

### 对话模式（claude.ai、ChatGPT 等）
1. 在 claude.ai 创建名为 **Job Vault** 的 Project，将 `SKILL.md` 上传至项目知识库
2. 新开对话，自然描述需求即可触发，无需显式调用，例如：
   - "帮我整理一下我的项目经历准备面试"
   - "我有一个新 JD，帮我改一下陈述稿"
   - "我们来 mock 一下"
3. 每次对话结束后，将 AI 输出的可粘贴片段更新进 Job Vault 对应文档

### Agent 模式（有文件读写权限的 AI 工具，如 Claude Code）
1. 本地创建 `Job Vault/` 文件夹，放入三份文档
2. 安装 `interview-project-prep.skill`
3. AI 在每个阶段确认后自动写入文档，无需手动维护，迭代闭环完全自动化

## 文件说明

| 文件 | 用途 |
|------|------|
| `SKILL.md` | 对话模式使用，上传至 AI 知识库 |
| `interview-project-prep.skill` | Agent 模式使用，直接安装 |

## 迭代说明

Skill 初版在 Claude 上完成设计，随后在其他 AI 平台（ChatGPT）进行了真实场景验证，根据实际使用中发现的问题回来迭代——包括沉淀时机判断、阶段间衔接路口、元项目底稿结构、输出模板格式等，均来自真实使用反馈而非假设推导。当前版本保持平台通用性，适配对话模式和 Agent 模式两种运行环境，持续迭代中。
